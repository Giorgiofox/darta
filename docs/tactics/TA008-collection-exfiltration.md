# TA008 — Collection and Exfiltration

The adversary intercepts, captures, or extracts data from UAS systems including live video feeds, telemetry streams, stored flight plans, sensor outputs, and mission intelligence data. UAS platforms routinely carry high-value data: imagery of sensitive facilities, precision mapping data, real-time surveillance feeds, and operational planning information.

---

### T008.001 — Video Feed Interception

**Platform scope**: Consumer, Enterprise, Military | **Minimum actor tier**: L1

**Description**: Passive or active interception of unencrypted or weakly encrypted video downlinks transmitted from the drone to the GCS, providing the adversary with real-time situational awareness equivalent to the legitimate operator's view.

**Sub-techniques**:
- DJI OcuSync video downlink passive capture using commercially available receiver hardware
- Analog FPV feed capture using RTL-SDR or dedicated FPV receiver
- H.264 and H.265 encoded digital stream interception and real-time decoding
- Infrared and thermal imaging video feed interception for platforms carrying IR payloads

**Countermeasures**: CM-001 (Encrypted and Authenticated C2 Link — when video is carried on the same encrypted channel), CM-012 (Onboard Data Encryption).

**Notes**: Unencrypted analog FPV video feeds remain widespread in consumer and hobbyist use. DJI consumer products use encrypted video transmission in recent generations, but earlier models transmitted unencrypted video. Military EO/IR feeds are separately encrypted but may be intercepted via side-channel means.

---

### T008.002 — Telemetry Stream Interception

**Platform scope**: All | **Minimum actor tier**: L1

**Description**: Passive capture of MAVLink or proprietary telemetry streams transmitted from the drone to the GCS to extract real-time position, mission status, sensor readings, and system health data.

**Sub-techniques**:
- MAVLink telemetry passive RF capture revealing GPS position, altitude, speed, battery, and mode in real time
- Proprietary protocol telemetry capture following reverse engineering of encoding
- APRS-based telemetry interception for long-endurance platforms using amateur radio frequencies

**Countermeasures**: CM-001 (Encrypted and Authenticated C2 Link protecting telemetry channel).

---

### T008.003 — Mission Data Exfiltration

**Platform scope**: All | **Minimum actor tier**: L2

**Description**: Unauthorized extraction of stored flight plans, waypoint databases, survey outputs, and mission-specific files from onboard storage media or GCS-connected systems.

**Sub-techniques**:
- Physical SD card removal from unattended drone and content duplication
- GCS database and mission file exfiltration via network compromise
- Cloud-synchronized mission data access using stolen operator credentials
- Survey and mapping point cloud, orthomosaic, and 3D model data theft from connected processing systems

**Countermeasures**: CM-012 (Onboard Data Encryption), CM-008 (GCS Network Segmentation).

---

### T008.004 — Specialized Payload Data Interception

**Platform scope**: Enterprise, Military | **Minimum actor tier**: L2

**Description**: Interception or theft of data generated by specialized drone payloads including multispectral sensors, LiDAR systems, signals intelligence collection packages, or military EO/IR and synthetic aperture radar payloads.

**Sub-techniques**:
- Multispectral and hyperspectral imagery exfiltration revealing crop health, facility layout, or terrain analysis
- LiDAR point cloud data theft providing precise 3D mapping of target areas
- Signals intelligence payload data intercept from COMINT or ELINT collection drone
- Synthetic aperture radar data exfiltration from military reconnaissance platform

**Countermeasures**: CM-012 (Onboard Data Encryption), CM-001 (encrypted payload downlink).

---

### T008.005 — Side-Channel Data Extraction

**Platform scope**: All | **Minimum actor tier**: L3

**Description**: Extraction of sensitive cryptographic keys or mission data through analysis of power consumption, electromagnetic emissions, or computational timing characteristics of onboard electronic components, without requiring direct access to protected memory or storage.

**Sub-techniques**:
- Power analysis of onboard cryptographic operations to extract key material
- Electromagnetic leakage analysis of flight controller processor revealing sensitive computations
- Timing side-channel attack on authentication or key exchange protocol implementation

**Countermeasures**: CM-012 (hardware security module use for key operations), CM-007 (physical access control).

## Detection Opportunities — TA008

| Technique | Detection Method | Data Source |
|---|---|---|
| T008.001 | Unauthorized RF receivers detected near operational area | RF monitoring; physical security |
| T008.002 | Unauthorized packet capture devices on or near flight path | RF direction finding |
| T008.003 | Unexpected access to GCS file shares or cloud storage | Access log monitoring; DLP |
| T008.004 | Unauthorized physical access to drone payload bay | Physical security; tamper evidence |
| T008.005 | Physical proximity device detected near drone storage | Physical security monitoring |
